Processing

Processing is a simple programming environment that was created to make it easier to develop visually oriented applications with an emphasis on animation and providing users with instant feedback through interaction. The developers wanted a means to “sketch” ideas in code. As its capabilities have expanded over the past decade, Processing has come to be used for more advanced production-level work in addition to its sketching role. Originally built as a domain-specific extension to Java targeted towards artists and designers, Processing has evolved into a full-blown design and prototyping tool used for large-scale installation work, motion graphics, and complex data visualization.

Processing consists of:

  • The Processing Development Environment (PDE). This is the software that runs when you double-click the Processing icon. The PDE is an Integrated Development Environment (IDE) with a minimalist set of features designed as a simple introduction to programming or for testing one-off ideas.
  • A collection of functions (also referred to as commands or methods) that make up the “core” programming interface, or API, as well as several libraries that support more advanced features such as sending data over a network, reading live images from a webcam, and saving complex imagery in PDF format.
  • A language syntax, identical to Java but with a few modifications.

size(400, 400);

Funciones

Las funciones son llamadas también "procedimientos” o “métodos” o “subrutinas”. El término que usamos nosotros es el de funciones, sin embargo en otros lenguajes pueden referirse como procedimientos, en Java específicamente el término correcto es “método”.

Processing nos permite crear visuales interesantes con pocas líneas de código, pero conforme nos aproximamos a proyectos más complejos, tales como aplicaciones en red o procesamiento de imagen, necesitaremos escribir ya bastantes líneas, cientos.

Las funciones son el medio para tomar partes del programa y separarles en piezas modulares, facilitando la lectura del código y su revisión.

Las funciones tienen más a cargo que sólo romper un programa en pedazos. Los argumentos (o “parámetros”) son valores que se “pasan” a una función. Un argumento es una declaración simple de variable dentro del paréntesis de la definición en la función, la cual es una variable local.

Cosas para recordar

  • Debemos pasar el mismo número de parámetros que el definido en la función.
  • El parámetro debe ser del mismo tipo dato que el declarado.
  • El valor puede ser literal(20,16 ,34, etc), una variable(x,y,etc) o el resultado de una expresión(random(10), 8+3, etc).
  • Los argumentos son variables locales para una función y sólo son accesibles dentro de la función misma.

Clases

Es importante comprender los fundamentos de la programación, desde siempre hemos estado usando lo que hasta ahora vemos, lo nuevo es la manera de pensar y estructurar. Una nueva manera de organizar lo aprendido.

Esto es lo que se conoce como OOP Programación orientada a objetos (Object Oriented Programming)

Un objeto es una cosa que tiene propiedades y puede hacer cosas. Las propiedades en los objetos son lo que conocemos como variables y las cosas que puede hacer son las funciones.

Una clase es un template del objeto donde viene especificada información relacionada con ella y las instrucciones que se realizaran.

Una clase tiene:

  • Nombre de la clase - Es especificado por class, el nombre de la clase ComoQuierasQueSeLlame y luego brackets(los nombres de las clases tradicionalmente comienzan en mayúscula).
  • Informacion - La data es una colección de variables normalmente las llamamos instancias.
  • Constructor - Es una función especial dentro de la clase que crea la instancia del objeto en sí. Es donde damos las instrucciones de cómo inicializar el objeto. Siempre tiene el mismo nombre de la clase y va seguido de dos paracentesis.
  • Metodos - Estas son las funciones que vimos anteriormente, es donde se ejecutan instrucciones especificas.

Arreglos

Nos permite contener en un sólo elemento una lista de variables. Un programa requiere múltiples instancias de datos similares, en esos casos los arreglos nos vienen como anillo al dedo. El tener una lista de variables nos es útil de dos principales maneras:

  • la lista guarda los elementos y permite localizarlos
  • la lista guarda el orden de los mismos

Los arreglos se declaran de la misma manera que las variables, sólo que se ven un tanto distintos por el uso de brackets vacíos

Una propiedad fundamental de los arreglos es que son de un tamaño específico, y una vez que definimos el tamaño no puede ser modificado

Creamos el arreglo en el momento que definimos el tamaño
